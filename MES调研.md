# MES系统调研



真实用户需求---- 避免过度规划 先慢慢来 慢慢迭代





MES不是一个系统，而是一个**能力集合**。你先要避免一个坑：

> ❌ 直接做“大而全 MES” → 失败概率极高
>  ✅ 从“制造能力平台”切入 → 逐步演进

### What（是什么）

**MES = 车间级执行系统（Shopfloor OS）**

核心作用：

- 管理生产执行过程（而非计划）
- 实时记录生产行为
- 打通设备 → 人 → 数据 → 系统

👉 本质：







Plain Text

MES = 数据采集系统 + 流程控制系统 + 可追溯系统

``

显示更多行

典型能力：

- 工单执行
- 工艺路线
- 工站管理
- 数据采集
- 质量/SPC
- 追溯

📌 关键一句：

> MES 的本质是“记录和约束生产行为”，而不是管理业务计划

------

###  Why（为什么需要）

MES存在的根因：

#### ✅ 工业三大问题：

| 问题         | 说明                      |
| ------------ | ------------------------- |
| 信息断层     | ERP不懂产线，设备不懂业务 |
| 过程不可控   | 产线靠人经验              |
| 数据不可追溯 | 出了问题找不到原因        |

------

#### ✅ MES解决什么问题

- ✅ 生产透明化（谁/在哪/做什么）
- ✅ 质量可控（SPC / 检测）
- ✅ 全链路追溯（SN → 工艺 → 数据）
- ✅ 实时决策（看板）

👉 结果：

- 提升效率
- 降低不良率
- 数据驱动决策

------

###  Who（谁用）

MES是一个**多角色系统**

| 角色       | 使用方式               |
| ---------- | ---------------------- |
| 操作员     | 扫码 / 过站 / 输入数据 |
| 工艺工程师 | 定义工艺流程           |
| 质量工程师 | SPC / 检测             |
| 设备工程师 | 查看设备数据           |
| 管理层     | 看报表 / KPI           |

👉 设计重点：

> ✅ UI必须区分角色，不是一个系统一个页面

------

| -------- | ---- |

| 运维     | 10%  |

👉 关键结论：

> ❗ MES最难的是“落地”，不是写代码
